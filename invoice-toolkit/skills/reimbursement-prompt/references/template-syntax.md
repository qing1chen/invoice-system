# 模板语法详解

## 区段类型

| 类型 | 标记 | 说明 |
|------|------|------|
| 一次性区段 | 无标记（默认） | 仅执行一次，用第一条记录的数据渲染 |
| 重复区段 | `===标题===` 开始，`======` 结束 | 为每条记录展开一次 |

## 变量引用

用 `{{变量名}}` 引用数据字段。

### 报销记录字段（records 表）

`db_id`, `序号`, `姓名/公司`, `填写日期`, `金额`, `物品简介`, `备注`, `类别`, `附件路径`, `匹配附件`

### 发票信息字段（invoices 表，经匹配关联）

`发票号码`, `价税合计`, `商品名称`, `商品单价`, `开票日期`, `销售方名称`,
`发票类型`, `发票路径`, `购方名称`, `购方税号`, `税额`, `发票代码`, `校验码`,
`匹配姓名`, `匹配简介`, `匹配金额`

> 如需引用新字段，只需在模板中写 `{{字段名}}`，系统会自动从数据库读取（只要 invoices 表中有该列且值非空）。

### 计算字段（由 `calculateTransferAmounts` 注入）

| 字段 | 说明 | 用法 |
|------|------|------|
| `转卡金额` | 该户名下所有选中记录的金额累加总和 | 重复区段中使用 |
| `转卡明细` | 金额明细字符串，如 "100.00+200.00=300.00" | 重复区段中使用 |
| `转卡汇总` | 去重后的完整转卡指令文本，每个户名只出现一次 | **一次性区段**中使用 |
| `附件汇总` | 所有需上传附件的完整指令文本，按附件路径去重 | **一次性区段**中使用 |
| `记录ID映射` | 所有选中记录的 `db_id ↔ 发票号码/姓名/金额` 对应表 | **模板最开头**使用 |

### 金额计算逻辑

1. 按 `姓名/公司` 分组，对同组内所有选中记录的 `金额` 求和
2. 每个户名只生成一条转卡指令（去重）
3. 多笔金额时附带加法明细
4. `转卡汇总` 和 `附件汇总` 是一次性字段，放在模板的 once 区段中

渲染示例：
```
在「转卡」区域中找到户名为张三的那一行，将其金额设置为 350.00（明细：100.00+150.00+100.00=350.00），点击保存。
在「转卡」区域中找到户名为李四的那一行，将其金额设置为 200.00，点击保存。
```

### 附件汇总与去重逻辑

**核心问题**：多条记录可能指向同一个附件文件（尤其加班餐场景——多张发票合并生成一份情况说明，每条记录的 `附件路径` 都指向同一个文件）。

**去重规则**：
1. 按 `附件路径` 字段的完整路径值去重
2. 同一路径只生成一条上传指令
3. 发票文件（`发票路径`）在重复区段中逐条处理，不纳入附件汇总
4. 格式转换后的路径（如 `.docx` → `.pdf`）以转换后的最终路径为准
5. 路径归一化：统一分隔符、去除尾部斜杠

**典型场景**：
```
记录1: 张三, 32元加班餐 → 附件路径=/app/output/张三+94+加班餐报销说明.pdf
记录2: 张三, 62元加班餐 → 附件路径=/app/output/张三+94+加班餐报销说明.pdf（同一文件）
→ 附件汇总只输出一条上传指令
```

## 操作指令书写规范

模板中所有可交互元素必须明确标注类型：

| 元素类型 | 书写格式 | 示例 |
|----------|----------|------|
| 按钮 | 点击「名称」按钮 | 点击「保存」按钮 |
| 链接 | 点击「名称」链接 | 点击「点击修改信息」链接 |
| 输入框 | 在「标签」输入框中填写 XXX | 在「手机」输入框中填写 138… |
| 下拉框 | 在「标签」下拉框中选择「值」 | 在「报销投递校区」下拉框中选择「千佛山校区」 |
| 复选框 | 点击该行对应的复选框将其选中 | — |

## record_id 注入

模板顶部使用 `{{记录ID映射}}` 占位符，渲染后展开为：
```
本报销单包含 3 条记录，record_id 与发票的对应关系如下：
  - record_id=12, 序号=1, 姓名=张三, 票号=12345678, 金额=94.00
  - record_id=15, 序号=2, 姓名=李四, 票号=87654321, 金额=200.00
如需跳过某条记录的操作，请在 skip 动作的 value 中注明对应的 record_id。
```

LLM 在开始执行前即可看到全部 ID，需要 skip 时自行从映射表中查找。
