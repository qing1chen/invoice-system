# 发票报销管理系统 — 前端模块化文档

> Invoice Toolkit Frontend · 基于 React + Vite · MCP Server 通信

---

## 1. 项目概述

本项目是山东大学课题组的发票报销管理系统前端，通过 **MCP (Model Context Protocol)** 协议与后端 Invoice Toolkit Server 通信，提供发票 OCR 识别、智能匹配、自动分类、政策 RAG 问答等功能。

前端界面的 **文件管理** 和 **报销明细** 页面已与后端项目目录 `D:\Spyder_down\llm\invoice-toolkit\data` 下的真实数据打通：

| 前端页面 | 对应后端数据 | 同步方式 |
|----------|-------------|----------|
| 文件管理 (FileManager) | `data/课题组成员文件/` 目录 | MCP 工具 `invoice_list_member_files` 实时拉取 |
| 报销明细 (TableEditor) | `data/明细.xlsx` | MCP 工具 `invoice_read_table` 读取 / `invoice_save_table` 回写 |

### 技术栈

| 技术 | 用途 |
|------|------|
| React 18 | UI 框架 |
| Vite | 构建工具 |
| MCP Protocol (Streamable HTTP) | 前后端通信 |
| localStorage | 客户端数据缓存（MCP 不可用时降级使用） |
| Noto Sans SC + JetBrains Mono | 字体方案 |

### 角色权限

| 角色 | 可用页面 |
|------|----------|
| 课题组成员 | 概览、我的文件、报销明细 |
| 管理员 | 全部页面（含工具控制台、成员管理、服务器配置） |

## 3. 分层架构说明

系统按 **四层架构** 组织，依赖关系自下而上单向流动：

```
┌─────────────────────────────────────────────────┐
│  Entry（入口层）                                  │
│  main.jsx → App.jsx → InvoiceApp.jsx            │
├─────────────────────────────────────────────────┤
│  Pages（页面层）                                  │
│  LoginPage / Sidebar / Dashboard / FileManager  │
│  TableEditor / ToolsConsole / MembersPanel      │
│  ServerSettings                                  │
├─────────────────────────────────────────────────┤
│  UI Components（组件层）                          │
│  Button / Badge / Card / Input / Spinner        │
│  EmptyState / PageHeader / CellEditor           │
├─────────────────────────────────────────────────┤
│  Foundation（基础层）                             │
│  constants/ │ theme/ │ services/ │ utils/        │
└─────────────────────────────────────────────────┘
```

**核心原则**：

- **上层依赖下层**：页面组件引用 UI 组件和基础层，反过来不允许
- **同层不互相依赖**：各页面组件之间完全独立，无交叉 import
- **基础层互相独立**：`constants`、`theme`、`services`、`utils` 各自独立（`services/data.js` 引用 `services/mcp.js`、`utils/helpers.js` 引用 `constants` 和 `theme` 是仅有的例外）

## 4. 各模块详细说明

### 4.1 基础层

#### `constants/index.js`

集中管理所有配置数据，修改业务规则只需编辑此文件。

| 导出名 | 类型 | 说明 |
|--------|------|------|
| `NAME_LIST` | `string[]` | 课题组全部成员姓名（29 人） |
| `CATEGORIES` | `string[]` | 报销类别：出差、加班餐、快递、打印、打车、材料、论文和专利 |
| `STATUS_LIST` | `string[]` | 状态枚举：待报销、审核中、已报销、已驳回 |
| `ADMIN_USER` | `object` | 管理员账号配置 `{ name, role, password }` |
| `TABLE_COLUMNS` | `string[]` | 报销明细表 8 个列名 |
| `DEFAULT_MCP_URL` | `string` | MCP Server 默认地址 `http://localhost:8000` |
| `MCP_TOOL_MAP` | `object` | 前端工具 id → MCP 工具名 + 默认参数的映射表（含数据同步工具） |
| `TOOLS` | `object[]` | 工具面板配置：id、名称、图标、描述、是否危险操作 |

**MCP_TOOL_MAP 中的数据同步工具**：

| 映射 key | MCP 工具名 | 用途 |
|----------|-----------|------|
| `listFiles` | `invoice_list_member_files` | FileManager 拉取成员目录文件 |
| `readTable` | `invoice_read_table` | TableEditor 读取 明细.xlsx |
| `saveTable` | `invoice_save_table` | TableEditor 回写 明细.xlsx |

**迭代指引**：
- 新增课题组成员 → 在 `NAME_LIST` 中添加姓名
- 新增报销类别 → 在 `CATEGORIES` 中添加
- 新增 MCP 工具 → 在 `MCP_TOOL_MAP` 和 `TOOLS` 中同步添加

#### `services/mcp.js`

封装与 MCP Server 的全部通信逻辑。每次工具调用会自动执行完整握手流程：`initialize` → `notifications/initialized` → `tools/call`。

| 导出函数 | 参数 | 返回 | 说明 |
|----------|------|------|------|
| `callMcpTool(toolName, args, serverUrl)` | 工具名、参数对象、服务器地址 | `Promise<object>` | 调用指定 MCP 工具 |
| `listMcpTools(serverUrl)` | 服务器地址 | `Promise<Array>` | 获取已注册工具列表 |
| `extractMcpText(result)` | 工具返回值 | `string` | 从返回结果中提取纯文本 |

**内部流程**：

```
callMcpTool("invoice_run_ocr", {})
  │
  ├── initMcpSession(serverUrl)
  │     ├── POST /mcp  { method: "initialize" }
  │     ├── 获取 mcp-session-id
  │     └── POST /mcp  { method: "notifications/initialized" }
  │
  └── POST /mcp  { method: "tools/call", params: { name, arguments } }
        └── parseMcpResponse()  ← 自动处理 JSON / SSE 两种格式
```

**迭代指引**：
- 切换 WebSocket 传输 → 修改 `initMcpSession` 和请求方式
- 添加认证 → 在 `MCP_HEADERS` 中添加 Authorization 头
- 添加请求重试 → 在 `callMcpTool` 中包装重试逻辑

#### `services/data.js`（新增）

封装前端与 MCP Server 之间的 **数据同步** 逻辑，是 FileManager 和 TableEditor 与后端真实数据交互的桥梁。

| 导出函数 | 参数 | 返回 | 说明 |
|----------|------|------|------|
| `fetchMemberFiles(serverUrl, member?)` | 服务器地址、可选成员名 | `Promise<{files, error?}>` | 从 `data/课题组成员文件/` 拉取文件列表 |
| `fetchTableData(serverUrl)` | 服务器地址 | `Promise<{rows, columns, error?}>` | 从 `data/明细.xlsx` 读取报销记录 |
| `saveTableData(serverUrl, rows)` | 服务器地址、记录数组 | `Promise<{success, message}>` | 将编辑后的数据回写到 `data/明细.xlsx` |

**降级策略**：当 MCP Server 不可用时，函数返回 `error` 字段，调用方回退到 localStorage 本地数据。

**数据同步流程**：

```
┌─ InvoiceApp 启动 ─┐
│                    │
│  1. 先加载 localStorage（保证即时可用）
│  2. 异步调用 fetchMemberFiles + fetchTableData
│     ├── 成功 → 用远程数据覆盖 state + 缓存到 localStorage
│     └── 失败 → 保持本地数据不变
│                    │
└────────────────────┘

┌─ TableEditor 保存 ─┐
│                     │
│  1. 用户点击「保存到 xlsx」
│  2. 调用 saveTableData → MCP invoice_save_table
│     ├── 后端备份原文件为 明细.xlsx.bak
│     └── 写入新数据到 明细.xlsx
│  3. 同时 saveData 缓存到 localStorage
│                     │
└─────────────────────┘
```

**迭代指引**：
- 新增数据同步维度 → 在此文件中添加新的 fetch/save 函数
- 添加乐观更新 → 先更新 state 再发请求，失败时回滚
- 添加变更检测 → 对比本地和远程数据的 hash 值

#### `services/storage.js`

提供统一的持久化读写接口，当前基于 `localStorage` 实现。作为 MCP 不可用时的降级数据源。

| 导出函数 | 参数 | 说明 |
|----------|------|------|
| `loadData(key, fallback)` | 键名、默认值 | 读取并解析 JSON，失败返回 fallback |
| `saveData(key, data)` | 键名、数据对象 | 序列化并写入 |

系统使用的存储键名：

| 键名 | 存储内容 |
|------|----------|
| `invoice-user` | 当前登录用户 `{ name, role }` |
| `invoice-files` | 文件列表（MCP 拉取后缓存 / 本地上传数据） |
| `invoice-table` | 报销明细表数据（MCP 拉取后缓存 / 本地编辑数据） |
| `invoice-mcp-url` | 自定义 MCP Server 地址 |

**迭代指引**：
- 迁移到 IndexedDB → 只需修改本文件的两个函数实现
- 接入后端 API 存储 → 改为 `async` 函数 + fetch 调用

### 4.2 UI 组件层 (`components/ui/`)

所有基础 UI 组件通过 `components/ui/index.js` 统一导出，页面组件按需引入：

```js
import { Button, Card, Badge } from "./ui";
```

```jsx
<Button variant="primary" size="md" disabled={false} onClick={fn}>
  提交

```

| 属性 | 可选值 | 默认值 | 说明 |
|------|--------|--------|------|

```jsx
<Badge color={T.success}>已完成</Badge>
```

| 属性 | 默认值 | 说明 |
|------|--------|------|

```jsx
<Card hover onClick={fn} style={{ padding: "16px" }}>
```

| 属性 | 说明 |

## 5. 数据流

```
                ┌───────────────────────┐
                │   MCP Server 后端      │
                │  data/课题组成员文件/   │  ← invoice_list_member_files
                │  data/明细.xlsx        │  ← invoice_read_table / invoice_save_table
                └──────────┬────────────┘
                           │ MCP Protocol
                    services/data.js
                           │
                    services/mcp.js
                           │
                ┌──────────▼────────────┐
                │  InvoiceApp (全局state) │
                │  ├── files            │ → Dashboard / FileManager / MembersPanel
                │  ├── tableData        │ → Dashboard / TableEditor / MembersPanel
                │  ├── dataSource       │ → FileManager / TableEditor
                │  └── mcpUrl           │ → FileManager / TableEditor / ToolsConsole / ServerSettings
                └──────────┬────────────┘
                           │
                    services/storage.js
                           │
                ┌──────────▼────────────┐
                │   localStorage 缓存    │  ← 降级数据源
                └───────────────────────┘
```

- `files` 和 `tableData` 通过 props 传递给子组件
- 子组件通过 `setFiles` / `setTableData` 回调更新全局状态
- 每次更新同时调用 `saveData()` 缓存到 localStorage
- FileManager / TableEditor 可通过按钮触发 `onRefresh` 手动刷新远程数据
- TableEditor 额外支持通过「保存到 xlsx」调用 `saveTableData()` 回写后端

5. 如有新的全局状态，在 `InvoiceApp.jsx` 中添加 `useState`

### 6.6 添加新 UI 组件的步骤

1. 在 `components/ui/` 下创建 `NewComponent.jsx`

3. 导出为 `default`
4. 在 `components/ui/index.js` 中添加 `export { default as NewComponent } from "./NewComponent"`
5. 在页面组件中按需引入 `import { NewComponent } from "./ui"`
